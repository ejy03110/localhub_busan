💬

실시간 경험 정보 부족

공공데이터만으로는 혼잡도, 방문 후기, 현장 분위기 같은 실제 경험을 알기 어렵습니다.

🔗

정보와 커뮤니티의 분리

장소 정보와 사용자 게시글이 따로 존재하여 원하는 정보를 여러 서비스에서 찾아야 합니다.

핵심 기능

커뮤니티를 중심으로 공공데이터·지도·통계를 연결합니다.

🗂️

익명 커뮤니티

회원가입 없이 게시글 목록·상세·작성·수정·삭제를 제공하고, 비밀번호로 수정·삭제 권한을 확인합니다.

📍

장소 연계 게시글

게시글 작성 시 관련 장소를 선택하고, 장소 상세 화면에서 관련 후기·질문을 함께 확인합니다.

🤖

지역정보 챗봇

제공된 부산 공공데이터를 바탕으로 장소 검색과 추천 질문에 답하고, 데이터에 없는 내용은 없다고 안내합니다.

1순위 · 게시판 추가 기능

검색, 조회수, 좋아요, 북마크, 태그를 적용하여 정보 소비와 참여를 강화합니다.

2순위 · 지도 시각화

장소 좌표를 마커로 표시하고 카테고리별 필터와 관련 게시글 연결을 제공합니다.

3순위 · 데이터 대시보드

카테고리별 장소 수와 게시글 현황을 시각화하여 서비스 내 정보 분포를 보여줍니다.

MVP 범위

3일 개발 기간을 고려해 핵심 범위를 명확히 제한합니다.

Must Have

  • 부산 데이터 기반 지역정보 제공
  • 익명 게시판 CRUD 및 비밀번호 권한
  • OpenAI 기반 지역정보 챗봇
  • Vue.js 3 SPA 및 localStorage
  • Netlify 배포와 기능명세서·WBS

Should Have

  • 게시글 검색·조회수·좋아요
  • 장소·게시글 연계 지도
  • 장소·게시글 데이터 대시보드

Won't Have

  • 회원가입·로그인
  • 부산 외 4개 권역
  • 경로 안내·날씨·실시간 알림
  • 다국어·축제 캘린더

팀 회의에서 확정할 항목

아래 항목에 합의하면 화면 설계와 역할 분담을 진행할 수 있습니다.

1. 게시글 카테고리 질문 / 후기 / 추천 / 지역소식 / 정보제보 중 어떤 항목을 사용할지 결정
2. 게시글과 장소 연결 방식 게시글 작성 시 관련 장소를 필수 또는 선택으로 지정할지 결정
3. 지도 표시 범위 관광지·음식점·문화시설 전체를 보여줄지, 핵심 카테고리만 보여줄지 결정
4. 대시보드 지표 카테고리별 장소 수 / 지역별 분포 / 게시글 수 / 좋아요 수 중 2개 정도 선정
5. 챗봇 답변 범위 공공데이터만 활용할지, localStorage 게시글 검색까지 포함할지 결정
