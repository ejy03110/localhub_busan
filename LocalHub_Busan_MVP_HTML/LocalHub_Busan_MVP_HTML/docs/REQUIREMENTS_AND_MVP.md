# LocalHub 요구사항 분석 및 MVP 정의

## 1. 선정 권역

부산

## 2. 핵심 문제

부산의 해변·관광지·축제·문화시설·맛집 공공데이터가 여러 항목으로 분산되어 있어 주민과 관광객이 정보를 빠르게 탐색하기 어렵고,
실제 이용 경험을 함께 확인하기 어렵다.

## 3. 타깃 사용자

- 부산을 방문하려는 관광객
- 부산 지역 생활·문화 정보를 찾는 주민
- 행사, 관광지, 맛집 정보를 간편하게 질문하려는 사용자

## 4. 성공 지표

- 사용자가 3회 이내의 클릭으로 원하는 지역 정보에 도달
- 게시글 작성·조회·수정·삭제 흐름 정상 동작
- 모바일 화면에서 주요 기능 이용 가능
- 챗봇에서 주요 질의 유형에 대한 응답 UI 제공

## 5. MoSCoW 요구사항

### Must have

1. Vue.js SPA로 구현 가능한 화면 구조
2. 제공 JSON 기반 지역 정보 표시 구조
3. 익명 커뮤니티 목록·상세·작성·수정·삭제
4. 수정·삭제용 비밀번호 확인
5. localStorage 기반 게시글 저장
6. 플로팅 챗봇 UI와 대화 히스토리
7. 모바일 반응형 화면
8. Netlify 정적 배포 가능한 구조

### Should have

1. 관광지·문화시설·축제·맛집 지도 시각화
2. 게시글 제목 및 카테고리 검색
3. 축제 일정 캘린더

### Could have

1. 현재 날씨와 여행 적합도 카드
2. 즐겨찾기 및 좋아요
3. 링크 공유
4. 다국어 지원

### Won't have

1. 회원가입 및 로그인
2. 서버 데이터베이스
3. 사용자 간 실시간 게시글 동기화
4. WebSocket 알림
5. 결제 및 예약
6. 관리자 권한 시스템

## 6. 화면 목록

| 화면 | 파일 | MVP 기능 |
|---|---|---|
| 홈 | index.html | 지역 소개, 카테고리, 최근 글, 챗봇 |
| 지도 탐색 | explore.html | 카테고리 필터, 지도 핀, 장소 카드 |
| 축제 캘린더 | festival.html | 날짜별 축제 시각화 |
| 게시판 목록 | board.html | 목록, 검색, 글쓰기 |
| 게시글 상세 | post-detail.html | 조회, 비밀번호 기반 수정·삭제 |
| 게시글 작성·수정 | post-write.html | 제목, 내용, 카테고리, 비밀번호 |
